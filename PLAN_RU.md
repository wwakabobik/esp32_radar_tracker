# Personal Presence Hub — План и ТЗ

ESP32 + HLK-LD2410C превращается из корпоративного шпиона в личный умный гаджет.

---

## Железо

| Компонент | Модель | Интерфейс |
|---|---|---|
| MCU | ESP32-D0WD-V3 (240 МГц, dual-core) | — |
| Радар | HLK-LD2410C | UART |
| Дисплей | 0.96" OLED SSD1306 (предположительно) | I2C |
| Кнопки | 2 шт, GPIO to GND | GPIO |
| Питание | USB (CP2102) или автономно | — |

> **Шаг 0 перед разработкой:** сфоткать carrier-плату сверху и определить реальные GPIO пины для дисплея, кнопок и UART радара.

---

## Режимы устройства

### 🖥 Work Mode (основной, рабочий день)

**Что делает девайс:**
- Трекает присутствие за столом через LD2410C
- Дисплей показывает: время текущей сессии + суммарно за день
- Кнопка 1 (короткое нажатие): пауза/возобновление трекинга
- Кнопка 1 (долгое нажатие): сброс текущей сессии
- Кнопка 2: переключить режим (Work → Sleep → Media → Work)

**Что делает демон на маке:**
- Хранит историю сессий в SQLite
- Отправляет уведомление "встань, прогуляйся" каждые N минут непрерывного сидения
- Шлёт на дисплей: время сессии, напоминания

### 🌙 Sleep Mode (ночью, девайс у кровати)

**Что делает девайс:**
- Радар в режиме высокой чувствительности — ловит микродвижения тела (дыхание ~15–18 вдохов/мин)
- Дисплей гаснет (или минимальная яркость)
- Кнопка 1: отметить "лёг спать" вручную
- Кнопка 2: отметить "проснулся" вручную

**Что делает демон на маке:**
- Анализирует паттерн сигнала: засыпание определяется по стабилизации дыхания
- Записывает: время засыпания, пробуждения, количество ночных движений
- Утром шлёт в Telegram сводку по сну

### 🎵 Media Mode (управление музыкой жестами)

**Что делает девайс:**
- Рука приближается быстро → следующий трек
- Рука удаляется быстро → предыдущий трек
- Рука зависает на определённом расстоянии → громкость пропорционально дистанции
- Дисплей: иконка режима + текущий трек (демон шлёт название)

**Что делает демон на маке:**
- Принимает жест-событие, выполняет AppleScript команду к музыкальному плееру
- Шлёт на дисплей название текущего трека

## MQTT топики

| Топик | Направление | Payload |
|---|---|---|
| `hub/radar` | ESP32 → демон | `{dist, s_energy, m_energy, presence, ts}` |
| `hub/mode` | двусторонний | `"work"` / `"sleep"` / `"media"` |
| `hub/button` | ESP32 → демон | `{id: 1\|2, event: "press"\|"long"}` |
| `hub/display` | демон → ESP32 | `{line1: "...", line2: "...", icon: "..."}` |
| `hub/gesture` | ESP32 → демон | `{type: "next"\|"prev"\|"vol", value: 0–100}` |
| `hub/ota/trigger` | демон → ESP32 | `{url: "http://mac-ip:8080/firmware.bin"}` |
| `hub/status` | ESP32 → демон | heartbeat каждые 30 сек |

## Telegram-бот — команды

```
/status        — режим, текущая сессия, онлайн ли девайс
/today         — часов за столом сегодня, количество перерывов
/week          — статистика по дням (текстом или график)
/sleep         — последняя ночь: лёг/встал/качество
/standup [N]   — напоминания каждые N минут (по умолчанию 45)
/mode <режим>  — сменить режим удалённо: work / sleep / media
/update        — залить новую прошивку по OTA
/settings      — текущие настройки
```

## Веб-интерфейс (localhost:8080)

- **Дашборд:** сегодня — время за столом, сессии, перерывы
- **История:** график присутствия по часам за неделю/месяц
- **Сон:** timeline ночей, тренды
- **Настройки:** порог напоминания, тихие часы, яркость дисплея

---

## OTA (обновление прошивки без USB)

1. Новый бинарник кладётся в папку демона
2. Команда `/update` в Telegram или кнопка в веб-интерфейсе
3. Демон шлёт MQTT `hub/ota/trigger` с URL
4. ESP32 скачивает бинарник с HTTP-сервера демона и перезаписывает себя
5. Перезагрузка, подтверждение в Telegram

## Технологический стек

| Слой | Технология |
|---|---|
| Прошивка | PlatformIO + Arduino framework (C++) |
| Протокол связи | MQTT |
| MQTT брокер | mosquitto (`brew install mosquitto`) |
| Демон | Python 3.12, asyncio, aiomqtt |
| Telegram | python-telegram-bot |
| Web API | FastAPI |
| Frontend | Простой HTML + Chart.js |
| База данных | SQLite |
| OTA HTTP | aiohttp |
| macOS управление | AppleScript (subprocess) |
